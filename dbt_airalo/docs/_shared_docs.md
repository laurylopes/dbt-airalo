{% docs user_id %}
Unique identifier for each user.
Sourced from production database user table.
{% enddocs %}

{% docs ip_country %}
Country code derived from user's IP address.
Format: ISO 3166-1 alpha-2 (e.g., US, GB, DE).
{% enddocs %}

{% docs platform %}
Platform through which user accesses the service.
Possible values: ios, android, web
{% enddocs %}

{% docs acquisition_channel %}
Marketing channel through which the user was acquired.
Examples: organic, paid, referral, social
{% enddocs %}

{% docs order_id %}
Unique identifier for each order transaction.
Sourced from production database order table.
{% enddocs %}

{% docs amount %}
Monetary amount of the transaction in original currency.
Positive values represent purchases, negative values represent refunds.
{% enddocs %}

{% docs currency %}
Currency code for the transaction amount.
Format: ISO 4217 (e.g., USD, GBP, EUR, JPY).
{% enddocs %}

{% docs esim_package %}
eSIM package or product purchased in this order.
Contains package identifier or SKU.
{% enddocs %}

{% docs payment_method %}
Payment method used for the transaction.
Examples: card, paypal, apple_pay, google_pay
{% enddocs %}

{% docs card_country %}
Country of the payment card used for the transaction.
Format: ISO 3166-1 alpha-2 (e.g., US, GB, DE).
{% enddocs %}

{% docs destination_country %}
Destination country for the eSIM package or service.
Format: ISO 3166-1 alpha-2 (e.g., US, GB, DE).
{% enddocs %}

{% docs status %}
Current status of the order in the order lifecycle.
Examples: pending, completed, cancelled, failed, refunded
{% enddocs %}

{% docs usd_rate %}
Exchange rate relative to USD as base currency.
Format: 1 USD = X units of the target currency.
Example: If USD/EUR = 0.85, then 1 USD = 0.85 EUR
{% enddocs %}

{% docs created_at %}
Timestamp when the record was originally created in the source system.
Format: UTC timestamp.
{% enddocs %}

{% docs updated_at %}
Timestamp when the record was last modified in the source system.
Format: UTC timestamp.
{% enddocs %}

{% docs completed_at %}
Timestamp when the order reached completed status.
NULL if order has not been completed.
{% enddocs %}

{% docs refunded_at %}
Timestamp when the order was refunded.
NULL if order has not been refunded.
{% enddocs %}

{% docs failed_at %}
Timestamp when the order failed.
NULL if order has not failed.
{% enddocs %}

{% docs dbt_scd_id %}

Format: Hash of unique_key + dbt_valid_from timestamp.
{% enddocs %}

{% docs dbt_updated_at %}
Timestamp when dbt snapshot detected and recorded this version of the record.
Used for auditing snapshot refresh history.
{% enddocs %}

{% docs valid_from %}
Timestamp when this version of the record became effective.
Used for point-in-time queries and SCD Type 2 tracking.
{% enddocs %}

{% docs dbt_valid_from %}
Timestamp when this version of the record became effective.
Used for point-in-time queries and SCD Type 2 tracking.
{% enddocs %}

{% docs valid_to %}
Timestamp when this version of the record was superseded by a new version.
NULL for current/active records.
Used for SCD Type 2 time-based filtering.
{% enddocs %}

{% docs dbt_valid_to %}
Timestamp when this version of the record was superseded by a new version.
NULL for current/active records.
Used for SCD Type 2 time-based filtering.
{% enddocs %}

{% docs is_current %}
Boolean flag indicating whether this is the current/active version of the record.
{% enddocs %}
